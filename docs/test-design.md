# 测试设计文档

## 1. 测试策略

本项目采用三层测试策略：
- **单元测试**：测试 Service、DAO、Provider 等业务逻辑，使用 Mock 隔离外部依赖
- **Widget 测试**：测试关键 UI 组件的渲染和交互
- **集成测试**：端到端测试核心用户流程（需要真实 WebDAV 服务器或本地测试服务器）

## 9. 测试工具和配置

```yaml
dev_dependencies:
  flutter_test:
    sdk: flutter
  mockito: ^5.4.4          # Mock 生成（@GenerateMocks 注解）
  build_runner: ^2.4.11    # 代码生成
  sqflite_ffi: ^2.3.3      # 单元测试中使用内存 SQLite
  fake_async: ^1.3.1       # 测试定时器（不需要真实等待 5 分钟）
```

**测试数据库：** 单元测试使用 `sqflite_ffi` 的内存数据库，每个测试用例 `setUp` 时独立初始化，`tearDown` 时销毁。

**定时器测试：** 使用 `fake_async` 包模拟时间流逝（`fakeAsync(() { ... clock.advance(...); })`），无需真实等待。

**Provider 测试：** 使用 `ProviderContainer` 加 mock 依赖测试 Riverpod Provider，不依赖 Flutter widget 树。

**Mock 对象清单：**
- `MockWebDAVClient` — 替代真实 WebDAV 请求
- `MockAudioHandler` — 替代 AudioHandler，验证 pause()/seek() 调用
- `MockSharedPreferences` — 隔离持久化读写
- `MockProgressDao` — 隔离数据库操作
- `MockConnectionDao` — 隔离连接存储操作
